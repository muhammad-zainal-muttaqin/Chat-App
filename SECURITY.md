# Security Policy

## Supported Versions

We provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We greatly appreciate the efforts of security researchers and the community to help keep Priva Chat secure.

**IMPORTANT**: Do not create a public issue for security vulnerabilities!

### How to Report

If you discover a security vulnerability, please:

1. **Email to**: security@privachat.app
   - Or create a [Security Advisory](https://github.com/muhammad-zainal-muttaqin/priva-chat/security/advisories/new) on GitHub (if you have access)

2. **Include the following information**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

3. **Timeline**:
   - We will respond within **48 hours**
   - We will provide updates every 7 days
   - We will inform you when the vulnerability is fixed

### Disclosure Policy

- We will acknowledge your report after the vulnerability is confirmed
- We will give credit to the reporter (if permitted)
- We will publish a security advisory after a patch is available
- We will provide a reasonable disclosure timeline

### Scope

Vulnerabilities that are in scope:

- ✅ End-to-end encryption bypass
- ✅ Authentication bypass
- ✅ SQL injection
- ✅ XSS (Cross-Site Scripting)
- ✅ CSRF (Cross-Site Request Forgery)
- ✅ Remote code execution
- ✅ Privilege escalation
- ✅ Data leakage
- ✅ Denial of Service (DoS)

Vulnerabilities that are **NOT** in scope:

- ❌ Social engineering
- ❌ Physical attacks
- ❌ DDoS attacks
- ❌ Spam
- ❌ Issues requiring physical access to device
- ❌ Issues requiring admin access to server

### Reward

We currently do not have a bug bounty program, but we greatly appreciate your contribution and will provide appropriate credit.

## Security Best Practices

For Priva Chat users:

1. **Use a strong password** (minimum 12 characters, combination of letters, numbers, symbols)
2. **Do not share private keys** with anyone
3. **Verify identity** of your conversation partner before sharing sensitive information
4. **Update the application** regularly
5. **Report suspicious activity** immediately

## Security Updates

Security updates will be announced through:
- GitHub Security Advisories
- Release notes
- Email notification (for critical vulnerabilities)

Thank you for helping keep Priva Chat secure!
